I/O LIBRARY

The I/O library provides file system operations including:
  • reading
  • writing
  • appending
  • existence checking
To use it, import !io in your program.
<import id="!io"/>

Function Reference

Name Arguments Returns Description
read path String Reads entire file as string. Errors if file not found.
read-bin path Array of integers Reads file as bytes. Each byte becomes an integer (0-255).
write path, data None Writes string to file (overwrites if exists, creates if not).
write-bin path, array None Writes byte array to file (overwrites if exists).
append path, data None Appends string to end of file. Creates file if not exists.
write_new path, data None Creates new file and writes data. Errors if file already exists.
exists path Boolean Returns true if file exists, false otherwise.

read

Reads the entire contents of a file as a string.
<assign id="content"> <call id="read"> <str>data.txt</str> </call> </assign> <call id="println"> <add> <str>File content: </str> <access id="content"/> </add> </call>

read-bin

Reads a file as binary data, returning an array of integers (0-255).
./image.png File size:

write

Writes a string to a file. Overwrites existing content.
Creates parent directories automatically.
<call id="write"> <str>output.txt</str> <str>Hello, World!</str> </call>

write-bin

Writes an array of integers as binary data to a file.
<assign id="data"> <arr> <int>72</int> <int>101</int> <int>108</int> <int>108</int> <int>111</int> </arr> </assign> <call id="write-bin"> <str>hello.bin</str> <access id="data"/> </call>

append

Appends data to the end of a file. Creates the file if it doesn't exist.
<call id="append"> <str>log.txt</str> <str>[INFO] Application started\n</str> </call> <call id="append"> <str>log.txt</str> <str>[INFO] User logged in\n</str> </call>

write_new

Creates a new file and writes data. Fails if the file already exists.
Similar to Python's 'x' file mode.
<!-- This succeeds if config.xee doesn't exist --> <call id="write_new"> <str>config.xee</str> <str><assign id="setting"><str>value</str></assign></str> </call> <!-- This fails with "File exists" error --> <call id="write_new"> <str>config.xee</str> <str>new content</str> </call>

exists

Checks whether a file exists before attempting to read it.
<if> <call id="exists"><str>data.csv</str></call> <block> <assign id="content"> <call id="read"><str>data.csv</str></call> </assign> <call id="println"><access id="content"/></call> </block> <block> <call id="println"><str>File not found</str></call> </block> </if>

Path Resolution

All paths are resolved relative to the current working directory
This is where the interpreter is executed,not relative to the script's location.
Tip: To use paths relative to your script cd into the script's directory first.
Alternative, use relative paths

Example:
cd /path/to/your/script xmlang-ee script.xee
Warning: Always check if a file exists before reading it.
The read function will crash if the file doesn't exist.
