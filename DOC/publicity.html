PUBLICITY & PRIVACY

XEE provides a module system based on file imports and variable visibility.
By default, all variables are private to the file they are defined in.
The <public> tag is used to explicitly expose variables to other files.

The publicity system in XEE provides:
  • <import> — imports variables from other files or built-in libraries
  • <public> — exposes a variable to other files
  • By default, variables are private (encapsulated)
  • Functions are variables too — they can be made public or kept private
  • No namespaces — public variables are added directly to the scope
Key Point: Variables are private by default.
You must explicitly mark them as <public> to make them accessible from other files.

Tags Reference

Tag Children Description
<import> 0 (self-closing) Imports a file or built-in library.
Built-in libraries are prefixed with ! (e.g., !std).
All public variables from the imported file are added to the current scope.
<public> 0 (self-closing) Publicizes a variable, making it available when this file is imported.
Variables without <public> are private.

How It Works

1. Private by Default

Any variable defined without a <public> tag is private to the file.
It cannot be accessed from other files, even if they import this file.
<!-- file: private.xee --> <assign id="secret"> <str>This is private</str> </assign> <!-- No public tag → secret is private -->

2. Making Variables Public

Use the <public> tag to expose a variable.
The tag must reference the variable by its id.
<!-- file: public.xee --> <assign id="config"> <dict> <pair><str>theme</str><str>dark</str></pair> <pair><str>language</str><str>en</str></pair> </dict> </assign> <public id="config"/> <!-- config is now public -->

3. Importing Files

Use <import> to load variables from other files.
Only public variables are imported — private variables remain inaccessible.
<!-- file: main.xee --> <import id="public.xee"/> <!-- Imports only public variables --> <call id="println"> <access id="config"/> <!-- config is available because it was public --> </call>

4. Built-in Libraries

Built-in libraries are imported with the ! prefix.
They contain public variables and functions that become available after import.
<import id="!std"/> <!-- Standard library --> <import id="!math"/> <!-- Math library --> <import id="!rand"/> <!-- Random library --> <import id="!io"/> <!-- I/O library --> <import id="!crypto"/> <!-- Crypto library -->

Functions as Public Variables

Functions are also variables in XEE.
They can be made public or kept private, just like any other variable.
<!-- file: math-utils.xee --> <!-- Private function (only usable inside this file) --> <func id="internal_helper"> <args><arg id="x"/></args> <block> <mul><access id="x"/><int>2</int></mul> </block> </func> <!-- No public → private --> <!-- Public function (usable after import) --> <func id="double"> <args><arg id="x"/></args> <block> <call id="internal_helper"> <access id="x"/> </call> </block> </func> <public id="double"/> <!-- double is public -->
<!-- file: main.xee --> <import id="math-utils.xee"/> <assign id="result"> <call id="double"> <int>5</int> </call> </assign> <!-- result = 10 -->

No Namespaces

XEE does not have namespaces. When you import a file, all public variables are added
directly to the current scope. This means you must be careful with name collisions.
Warning: If two imported files define public variables with the same name,
the later import will overwrite the earlier one.
<!-- ❌ Name collision --> <import id="file1.xee"/> <!-- defines public variable "config" --> <import id="file2.xee"/> <!-- also defines public variable "config" → overwrites --> <!-- ✅ Avoid name collisions by using unique names --> <!-- file1.xee defines "file1_config" --> <!-- file2.xee defines "file2_config" -->

Best Practices

  • Keep variables private unless they need to be shared
  • Use unique names for public variables to avoid collisions
  • Group related functions in a single file and publicize only the ones needed
  • Import only what you need — but since there are no namespaces, you import everything public
  • Functions are variables — publicize them if they should be callable from other files
Note: Since functions are variables, you can:
pass them around.
assign them to other variables
create anonymous functions.
This makes XEE a functional language in this regard.
