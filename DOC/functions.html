FUNCTIONS

Functions in XEE are defined using the <func> tag and called using the <call> tag.
Functions in XEE provide:
  • Named blocks of reusable code
  • Positional arguments (order matters)
  • Pure functional behavior (no side effects)
  • First-class functions (functions are variables)
  • Anonymous functions (functions without a name)
Key Point: Functions in XEE are pure
Meaning they cannot modify variables outside their own scope.
They only operate on their arguments and return a value.

Defining Functions

Basic Syntax

<func id="function_name"> <args> <!-- Argument definitions --> <arg id="arg1"/> <!-- First argument --> <arg id="arg2"/> <!-- Second argument --> <arg id="arg3"/> <!-- Third argument --> </args> <block> <!-- Function body --> <!-- Code that executes when called --> <!-- The last expression is returned --> </block> </func>

Function with No Arguments

<func id="greet"> <args/> <!-- Empty args = no arguments --> <block> <str>Hello, world!</str> </block> </func>

Function with Arguments

<func id="add"> <args> <arg id="a"/> <arg id="b"/> </args> <block> <add> <access id="a"/> <access id="b"/> </add> </block> </func>

Returning a Value

The last expression in the function body is automatically returned.
You can also use <return> to explicitly return a value.
<func id="max"> <args><arg id="a"/><arg id="b"/></args> <block> <if> <gt><access id="a"/><access id="b"/></gt> <access id="a"/> <!-- Return a --> <access id="b"/> <!-- Return b --> </if> </block> </func>

Calling Functions

Basic Syntax

Functions are called using the <call> tag with an id attribute.
Arguments are passed as children in the order they are defined.
<call id="function_name"> <arg1_value/> <!-- First argument --> <arg2_value/> <!-- Second argument --> <arg3_value/> <!-- Third argument --> </call>

Calling with No Arguments

<call id="flush"/> <!-- Self-closing — no arguments -->

Calling with Arguments

<assign id="result"> <call id="add"> <int>5</int> <!-- a = 5 --> <int>3</int> <!-- b = 3 --> </call> </assign> <!-- result = 8 -->

Calling with Mixed Types

<call id="greet_user"> <str>Alice</str> <!-- String argument --> <int>30</int> <!-- Integer argument --> <bool>true</bool> <!-- Boolean argument --> </call>

Argument Passing

Arguments in XEE are positional — the order they are passed in must match
the order they are defined in the <args> block.
Important: The order of arguments matters.
Passing arguments in the wrong order will produce incorrect results or errors.
<!-- Function defined with two arguments --> <func id="divide"> <args> <arg id="numerator"/> <!-- First argument --> <arg id="denominator"/> <!-- Second argument --> </args> <block> <div> <access id="numerator"/> <access id="denominator"/> </div> </block> </func> <!-- Correct call: numerator=10, denominator=2 --> <call id="divide"> <int>10</int> <int>2</int> </call> <!-- Result: 5 --> <!-- Incorrect call: numerator=2, denominator=10 --> <call id="divide"> <int>2</int> <int>10</int> </call> <!-- Result: 0 (wrong!) -->

Pure Functions

Functions in XEE are pure — they cannot modify variables outside their scope.
<assign id="x"><int>10</int></assign> <func id="try_modify"> <args><arg id="val"/></args> <block> <assign id="x"><int>999</int></assign> <!-- This is a NEW variable in the function scope --> <add><access id="val"/><int>5</int></add> </block> </func> <call id="try_modify"><int>20</int></call> <!-- x is STILL 10 — the function did not modify the outer variable -->
Note: This pure functional behavior means functions have no side effects.
To change state, you must explicitly return values and reassign them.

Anonymous Functions

Functions can be defined without an id, creating an anonymous function.
These are useful for temporary operations or when passing functions as arguments.
<!-- Anonymous function assigned to a variable --> <assign id="multiplier"> <func> <args><arg id="x"/><arg id="y"/></args> <block> <mul><access id="x"/><access id="y"/></mul> </block> </func> </assign> <call id="multiplier"> <int>4</int> <int>7</int> </call> <!-- Result: 28 -->

Functions as Variables

Functions are first-class citizens in XEE. They can be assigned to:
. variables passed as arguments
returned from other functions.
<!-- Define a function --> <func id="add_one"> <args><arg id="x"/></args> <block> <add><access id="x"/><int>1</int></add> </block> </func> <!-- Assign the function to another variable --> <assign id="increment"> <access id="add_one"/> </assign> <!-- Call the function through the new variable --> <call id="increment"> <int>5</int> </call> <!-- Result: 6 -->

Public Functions

Functions can be made public using the <public> tag.
<!-- file: utils.xee --> <func id="double"> <args><arg id="x"/></args> <block> <mul><access id="x"/><int>2</int></mul> </block> </func> <public id="double"/> <!-- double is now public --> <!-- file: main.xee --> <import id="utils.xee"/> <call id="double"><int>5</int></call> <!-- Result: 10 -->

Best Practices

  • Name functions clearly — use descriptive names that indicate what they do
  • Keep functions small — a function should do one thing and do it well
  • Use arguments — avoid relying on external variables (pure functions)
  • Document functions — especially public ones that others will use
  • Return values explicitly — use the last expression or <return>
  • Use anonymous functions for temporary or one-off operations
  • Make functions public only when they need to be used from other files
