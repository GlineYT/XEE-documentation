ID'S IN XEE

XMLang extended edition uses XML tags for syntax, thus, id's are also a part of the syntax

ID's in XEE serve primarily for identification of variables,functions or libraries
for example:
<import id="!std"/> Hello world <access id="a"/>
To better illustrate how each id is used, let's do a line-by-line analysis
and compare with python.
After the opening program tag, the second line is the "import" tag
<import id="!std"/>
When importing a library with the relevant tag, the id property is being used to identify which library is imported
The equivalent syntax in python would be:
import library
In this case the equivalent to the "import" statement is the tag, while the argument after it is the id
The second part is the variable assignment:
Hello world
The assign keyword serves to specify the name "id" and type.
The ID is the name of the variable, and the "str" tag inside specifies the type.
XEE is strongly typed, thus variable declaration and assignment behaves like C/C++.
Due to the strongly typed nature of XEE, a comparison to java would be more appropirate
String name = "John";
The assign tag functions similarly to the equal
And the id property would be equivalent to the "name" after the "String" declaration
The "Hello world" inside the str tag serves as the value. Since XEE is strongly typed, type must be declared
The next part is the function call
<access id="a"/>
The call tag calls the function named in the id property
Inside the function,as a child the variable is passed. See the "functions" section of the documentation
Because variables can't be directly accessed, but must be accessed with the access tag,
the variable to acess is named using the id property. The id must be exactly the name of the variable as it was assigned
The Python equivalent would be:
print(a)
In python, functions are called like keywords, and the arguments are in parentheses
while the variable to be accessed is directly entered
