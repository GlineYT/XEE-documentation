BINARY OPERATORS

Binary operators are tags that take exactly two children and perform an operation on them.
Unlike function calls, these are built-in language tags that operate directly on values.
Binary operators in XEE provide:
  • Arithmetic operations (+, -, *, /, etc.)
  • Bitwise operations (AND, OR, XOR, shifts)
  • Comparison operations (eq, ne, lt, gt, leq, geq)
  • Array, set, and dictionary operations
Note: Advanced mathematical functions are found in the Math Library as function calls.

Arithmetic Operators

Tag Operation Description
<add> a + b Adds two numbers (casts to float if needed).
If array/set: appends second value.
If strings: concatenates.
<sub> a - b Subtracts two numbers (casts to float if needed).
If array + integer: removes element at index.
If set: removes second value.
If dict: removes key-value pair by key.
<mul> a × b Multiplies two numbers (casts to float if needed).
<div> a ÷ b Divides two numbers. Errors if b = 0.
<mod> a % b Remainder of division. Errors if b = 0.
<pow> a^b Raises a to power of b. Negative exponent returns float.

Bitwise Operators

Tag Operation Description
<and> a & b Bitwise AND of integers/booleans.
If sets: returns intersection.
<or> a | b Bitwise OR of integers/booleans.
If sets: returns union.
<xor> a ^ b Bitwise XOR of integers/booleans.
If sets: returns symmetric difference.
<shl> a << b Shift left. Shifts bits of a left by b positions.
<shr> a >> b Shift right. Shifts bits of a right by b positions.

Comparison Operators

Tag Operation Description
<eq> a == b Returns true if values are equal.
<ne> a != b Returns true if values are not equal.
<lt> a < b Returns true if a is less than b.
<gt> a > b Returns true if a is greater than b.
<leq> a ≤ b Returns true if a is less than or equal to b.
<geq> a ≥ b Returns true if a is greater than or equal to b.

Arithmetic

Array and Set Operations

<!-- Append to array (nests the second array) --> <assign id="nested"> <add> <arr><int>1</int><int>2</int></arr> <arr><int>3</int><int>4</int></arr> </add> </assign> <!-- Result: [1, 2, [3, 4]] --> <!-- Set intersection --> <assign id="common"> <and> <set><int>1</int><int>2</int><int>3</int></set> <set><int>2</int><int>3</int><int>4</int></set> </and> </assign> <!-- Result: {2, 3} -->

Dictionary Removal

<assign id="dict"> <dict> <pair><str>name</str><str>Gameline</str></pair> <pair><str>age</str><int>25</int></pair> </dict> </assign> <!-- Remove a key-value pair --> <assign id="without_age"> <sub> <access id="dict"/> <str>age</str> </sub> </assign>
